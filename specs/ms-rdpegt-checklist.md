## 1. Wire Format — `MAPPED_GEOMETRY_PACKET` (MS-RDPEGT §2.2.1)

All multi-byte integers are **little-endian**.

### 1a. Fixed Header (24 bytes, all PDU types)

| Field              | Type    | Offset | Size |
|--------------------|---------|--------|------|
| `cbGeometryData`   | u32 LE  | 0      | 4    |
| `Version`          | u32 LE  | 4      | 4    | MUST be `0x00000001` |
| `MappingId`        | u64 LE  | 8      | 8    |
| `UpdateType`       | u32 LE  | 16     | 4    | 1=UPDATE, 2=CLEAR |
| `Flags`            | u32 LE  | 20     | 4    | Reserved (0) |

### 1b. GEOMETRY_UPDATE Extension (offset 24+, total min 73 bytes)

| Field                | Type    | Offset | Size |
|----------------------|---------|--------|------|
| `TopLevelId`         | u64 LE  | 24     | 8    |
| `Left`               | i32 LE  | 32     | 4    |
| `Top`                | i32 LE  | 36     | 4    |
| `Right`              | i32 LE  | 40     | 4    |
| `Bottom`             | i32 LE  | 44     | 4    |
| `TopLevelLeft`       | i32 LE  | 48     | 4    |
| `TopLevelTop`        | i32 LE  | 52     | 4    |
| `TopLevelRight`      | i32 LE  | 56     | 4    |
| `TopLevelBottom`     | i32 LE  | 60     | 4    |
| `GeometryType`       | u32 LE  | 64     | 4    | MUST be `0x00000002` |
| `cbGeometryBuffer`   | u32 LE  | 68     | 4    |
| `pGeometryBuffer`    | u8[]    | 72     | var  |

- GEOMETRY_UPDATE minimum: `72 + 32 = 104` bytes (RGNDATAHEADER + 0 rects).
- GEOMETRY_CLEAR total: `24` bytes.
- `cbGeometryData == 72 + cbGeometryBuffer` for UPDATE.
- **Correction**: Earlier draft listed a trailing `Reserved` byte; verified against the §4.1 120-byte example (cbGeometryData=120, 24+48+48=120) that no such byte exists.

## 2. Wire Format — RGNDATA Geometry Buffer

### 2a. `RGNDATAHEADER` (32 bytes, referencing [MS-WMF] §2.2.2.4)

| Field            | Type    | Size | Valid |
|------------------|---------|------|-------|
| `dwSize`         | u32 LE  | 4    | MUST be 32 |
| `iType`          | u32 LE  | 4    | MUST be 1 (RDH_RECTANGLES) |
| `nCount`         | u32 LE  | 4    | |
| `nRgnSize`       | u32 LE  | 4    | Reserved, don't validate |
| `rcBound.left`   | i32 LE  | 4    | |
| `rcBound.top`    | i32 LE  | 4    | |
| `rcBound.right`  | i32 LE  | 4    | |
| `rcBound.bottom` | i32 LE  | 4    | |

### 2b. `RECT` entries (16 bytes each, nCount array)

- `left:i32, top:i32, right:i32, bottom:i32` (LE)
- `cbGeometryBuffer == 32 + nCount * 16`

## 4. State Machine

Server-to-client only. Client opens DVC and receives; never sends.

```
[Closed] --DYNVC_CREATE--> [Open]
  Open:
    UPDATE (new id) → insert
    UPDATE (existing id) → replace
    CLEAR (existing id) → remove
    CLEAR (unknown id) → silent ignore
  Open --DYNVC_CLOSE--> [Closed] (flush all)
```

- `RdpegtClient` holds `BTreeMap<u64, GeometryEntry>`.
- `GeometryLookup` trait for consumers (RDPEVOR).
- Client never sends on this DVC.

## 5. Validation Rules

| Condition | Action |
|-----------|--------|
| `cbGeometryData < 24` | DecodeError |
| `Version != 1` | DecodeError |
| `UpdateType` not in {1,2} | DecodeError |
| UPDATE: `cbGeometryData != 72 + cbGeometryBuffer` | DecodeError |
| `GeometryType != 2` | DecodeError |
| `RGNDATAHEADER.dwSize != 32` | DecodeError |
| `RGNDATAHEADER.iType != 1` | DecodeError |
| `cbGeometryBuffer != 32 + nCount*16` | DecodeError |
| `nCount` × 16 overflow | DecodeError (checked_mul) |
| `nCount > MAX_RECTS_PER_GEOMETRY` | DecodeError |
| Active map exceeds `MAX_ACTIVE_MAPPINGS` | DecodeError |
| CLEAR for unknown MappingId | silent ignore |

## 6. DoS Caps (implementation-defined)

- `MAX_RECTS_PER_GEOMETRY = 4096`
- `MAX_ACTIVE_MAPPINGS = 1024`
- `MAX_CBGEOMETRYBUFFER = 65536`
- `MAX_CBGEOMETRYDATA = 65608` (72 + 65536)
